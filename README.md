# 📖 AI 故事创作助手

> 草稿 → 节拍 → 章节 → 大纲 — AI 辅助的全流程写作工具

一个面向**非专业写作者**的故事创作工具。你只需要写下脑海中的故事想法，AI 会帮你理清思路、规划结构、生成正文、跟踪大纲。不需要任何写作经验。

## ✨ 功能概览

### 四级管道

```
📥 草稿箱  ──→  📅 节拍板  ──→  📖 章节  ──→  📋 大纲
 自由写作       结构规划       正文创作       全局概览
```

每一级都有专属的工作面板和 AI 辅助工具：

| 层级 | 面板 | AI 工具 |
|:----|:----|:----|
| **草稿箱** | ✍️ 自由写作面板（暖色笔记本风格） | 续写思路、脑暴扩展、提炼核心、改写润色 |
| **节拍板** | 📐 结构规划面板（蓝色架构风格） | 节奏分析、逻辑检查、补充节拍 |
| **章节** | ✍️ 创作面板（紫色编辑器风格） | 续写/导演/脑暴/检查 4 种模式、3 种润色风格 |
| **大纲** | 📊 概览面板（绿色仪表盘风格） | 故事弧线分析、一致性检查 |

### 🎭 角色管理

独立的 **「👥 角色」面板**：

- **关系图谱** — 圆形 SVG 布局，角色节点按类型着色（⭐主角/🔹配角/💀反派）
- **角色档案卡** — 查看/编辑角色描述、目标、背景
- **关系编辑器** — 定义角色间的关系（如：师徒、恋人、母女）
- 创建故事时 AI 自动识别角色

### ✍️ 章节写作

- **4 种写作模式**：续写、导演（给指令）、脑暴（建议方向）、检查（分析优缺点）
- **3 种润色风格**：画面感、感人、紧张
- **阶段流**：📝草稿 → 👀初稿 → 🔧修改 → ✅审核 → 📖发布
- 章节关联节拍显示
- AI 生成计时器

### 💡 AI 辅助

- 创建故事后 AI 自动追问，完善设定（可点击问题快速回答）
- 每个层级都有专门的 AI 分析工具
- 自动提取草稿标题和章节梗概
- 根据已写章节自动更新大纲

## 🚀 快速开始

### 前置要求

- Python 3.10+
- DeepSeek API key（或其他兼容的 LLM API）

### 安装与运行

```bash
# 1. 进入项目目录
cd ~/hermes_workspace/story_tool

# 2. 安装依赖
pip install fastapi uvicorn

# 3. 配置 API 密钥（在 .env 或环境变量中）
export DEEPSEEK_API_KEY="sk-your-key"

# 4. 启动服务
python3 web_app.py

# 5. 打开浏览器
# http://localhost:8888
```

### 环境变量

| 变量 | 说明 |
|:----|:----|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥（默认后端） |
| `ZHIPU_API_KEY` / `GLM_API_KEY` | 智谱 API（备选后端） |

如需换模型，修改 `llm.py` 中的 API 调用配置。

## 🧭 使用流程

### 第 1 步：写下想法

在开始页面输入你的故事想法——可以是一段剧情、几个关键词、一个场景或角色设定。AI 会自动分析并提取角色、节拍。

### 第 2 步：自由写作（草稿箱）

在草稿箱自由书写，不要考虑格式。可以有多份草稿。AI 可帮你续写思路、脑暴扩展、提炼核心。

### 第 3 步：规划结构（节拍板）

勾选草稿 → 生成节拍。AI 分析节奏、检查逻辑漏洞、建议补充缺失环节。

### 第 4 步：正文创作（章节）

选择节拍 → 生成章节正文。使用续写/导演/脑暴/检查 4 种 AI 模式辅助写作。

### 第 5 步：全局审视（大纲）

自动同步已写章节，分析故事弧线，检查一致性。

## 🏗️ 技术架构

```
story_tool/
├── web_app.py           # FastAPI 后端（API + 静态文件服务）
├── framework.py         # 数据模型（Draft/Beat/Chapter/Character）
├── llm.py               # LLM API 调用封装
├── comprehension.py     # 故事理解和框架构建
├── storage.py           # 旧的存储层（兼容）
├── story.py             # 旧的 CLI 原型
├── start_server.sh      # 启动脚本
├── frontend/
│   └── index.html       # 单页 Web UI（纯 HTML/CSS/JS）
└── stories/             # 用户故事数据（JSON，已加入 .gitignore）
```

## 🗂️ 数据模型

```
StoryFramework
├── title / genre / tone
├── characters[]       ← 角色（含关系）
│   ├── name / role / description
│   └── relationships[]  ← {with_name, type, description}
├── drafts[]           ← 草稿（自由写作）
├── beats[]            ← 节拍（支持父子层级）
│   ├── title / description / location / characters
│   └── parent_beat_id  ← 次节拍关联
├── chapters[]         ← 章节（正文）
│   ├── title / prose / summary / beat_ids
│   └── stage ← 草稿→初稿→修改→审核→发布
└── outline (自动生成)
```

所有数据保存为 JSON 文件在 `stories/` 目录下。
